# SEC-006 — stdio-Transport für lokale Server

## Description

Lokale MCP-Server laufen mit den Privilegien des Users — sie haben Zugriff auf das Filesystem, Netzwerk-Interfaces, Subprozess-Spawning, Environment-Variablen, gespeicherte SSH-Keys, Browser-Cookies. Ein lokaler Server, der gleichzeitig auf einem Netzwerk-Port lauscht, weitet diese User-Privilegien auf jede Entity aus, die diesen Port erreichen kann.

Der Best-Practice-Standard im PDF (Sec 4.5) ist klar: lokale MCP-Server verwenden **ausschliesslich stdio-Transport**. Das eliminiert die Netzwerk-Angriffsfläche komplett. Kein offener Port, kein NeighborJack (siehe SEC-016), kein Remote-Access-Risiko bei kompromittiertem WLAN.

Dieser Check überlappt mit SEC-016 (0.0.0.0-Binding-Prevention) und SCALE-001 (Streamable HTTP für Cloud), aber mit anderem Fokus: SEC-006 fragt **nicht** «bindest du an 127.0.0.1 oder 0.0.0.0», sondern «bindest du überhaupt an einen Netzwerk-Port — obwohl du lokal läufst?».

Dual-Transport-Server mit `MCP_TRANSPORT=sse` für Cloud-Deployment sind okay — aber nur, wenn der lokale Default `stdio` ist und das Cloud-Setup explizit dokumentiert ist (siehe SCALE-001).

## Verification

### Modus 1: code_review (Default-Transport ist stdio)

```bash
grep -rE 'MCP_TRANSPORT|transport.*=.*stdio' src/
```

**Pass-Pattern:**

```python
# Default ist stdio, SSE/HTTP nur via expliziter ENV-Var
transport = os.environ.get("MCP_TRANSPORT", "stdio")
if transport not in ("stdio", "sse", "streamable-http"):
    raise ValueError(f"Unknown transport: {transport}")
mcp.run(transport=transport)
```

**Fail-Pattern:**

```python
# FAIL: hartcodiert auf SSE, auch lokal
mcp.run(transport="sse", host="0.0.0.0", port=8000)

# FAIL: Default ist SSE
transport = os.environ.get("MCP_TRANSPORT", "sse")
```

### Modus 2: documentation_check (README-Empfehlung)

```bash
grep -iE 'stdio|claude desktop|local' README.md README.de.md
```

**Pass:** README dokumentiert klar:

```markdown
## Local Usage

This server is designed for local use via stdio transport with Claude Desktop:

\```json
{
  "mcpServers": {
    "zh-education": {
      "command": "uvx",
      "args": ["zh-education-mcp"]
    }
  }
}
\```

For cloud deployment, see [Cloud Deployment](#cloud-deployment).
```

### Modus 3: runtime_test (Default-Start öffnet keinen Port)

```bash
# Server starten (ohne ENV-Var)
python -m my_mcp_server &
SERVER_PID=$!
sleep 2
# Prüfen: keine offenen TCP-Ports vom Prozess
ss -tlnp 2>/dev/null | grep $SERVER_PID
# Erwartet: keine Ausgabe — kein Port offen
kill $SERVER_PID
```

## Pass Criteria

- [ ] Default-Transport (ohne ENV-Var) ist `stdio`
- [ ] SSE/HTTP-Transport nur via expliziter `MCP_TRANSPORT`-ENV-Var aktivierbar
- [ ] README dokumentiert lokalen Use-Case primär (stdio + Claude Desktop)
- [ ] README dokumentiert Cloud-Use-Case separat mit Sicherheitshinweisen
- [ ] Default-Start öffnet keinen TCP-Port (Runtime-Test)

## Common Failures

| Anti-Pattern | Risiko |
|---|---|
| Default-Transport ist SSE/HTTP | Lokaler Server öffnet Port, NeighborJack-Risiko |
| README zeigt nur Cloud-Deployment | User vergessen lokale stdio-Option |
| `mcp.run(transport="sse", host="0.0.0.0")` hartcodiert | Maximales Risiko |
| Tutorial-Code mit `transport="sse"` als Beispiel | User kopieren ohne Verständnis |

## Remediation

```diff
+ import os

  if __name__ == "__main__":
-     mcp.run(transport="sse", host="0.0.0.0", port=8000)
+     transport = os.environ.get("MCP_TRANSPORT", "stdio")
+     if transport == "stdio":
+         mcp.run(transport="stdio")
+     elif transport in ("sse", "streamable-http"):
+         mcp.settings.host = os.environ.get("MCP_HOST", "127.0.0.1")
+         mcp.settings.port = int(os.environ.get("MCP_PORT", "8000"))
+         mcp.run(transport=transport)
+     else:
+         raise ValueError(f"Unknown transport: {transport}")
```

Plus README-Sektion:

```markdown
## Transport

This server defaults to **stdio** for local use. Only set `MCP_TRANSPORT=sse`
or `MCP_TRANSPORT=streamable-http` for cloud deployment, where the network
attack surface is mitigated by deployment infrastructure.

**Never** use HTTP/SSE transport on a local development machine — see
[security notes](docs/security.md).
```
