# CH-003 — Lehrpersonen-Einwilligung bei Volksschule-Daten

## Description

Personendaten von Lehrpersonen der Volksschule sind durch das Volksschulgesetz Kanton Zürich (VSG) und die Personalgesetze der Stadt Zürich besonders geschützt. Wenn ein MCP-Server diese Daten verarbeitet — sei es zur Statistik, zur internen Analyse oder zur Zuteilung — gelten zusätzliche Anforderungen über CH-002 hinaus:

1. **Klare Zweckbindung:** Daten dürfen nur für den im Verarbeitungsverzeichnis genannten Zweck verwendet werden. Sekundärnutzung (z.B. interne Auswertungen über Krankheitstage) ist meldepflichtig.

2. **Information & Einwilligung:** Lehrpersonen müssen über die Verarbeitung informiert sein. Bei nicht-notwendigen Verarbeitungen (jenseits Aufgabenerfüllung) ist die Einwilligung explizit einzuholen.

3. **Berufsverbands-Konsultation:** Bei systematischer Verarbeitung sollte die LCH-Sektion Stadt Zürich oder ZLV (Zürcher Lehrerinnen- und Lehrerverband) informiert sein — nicht zwingend rechtlich, aber best-practice für Sozialpartnerschaft.

4. **Zugriffsrechte:** Lehrpersonen haben jederzeit Auskunftsrecht (Art. 25 revDSG) und Berichtigungsrecht (Art. 32). Der Server muss diese Rechte technisch unterstützen können.

Beispiele für Volksschul-Daten im MCP-Kontext:
- Klassenzuteilung pro Lehrperson
- Pensum-Veränderungen
- Schulleitungs-Zuteilung
- Weiterbildungs-Daten

## Verification

### Modus 1: documentation_check (Information der Lehrpersonen)

```bash
find . -iname '*einwilligung*' -o -iname '*lehrperson*' -o -iname '*information-volksschule*'
```

**Pass:** Im Repo dokumentiert:
- Wie Lehrpersonen über die Verarbeitung informiert werden (Mailing, Intranet, Personalreglement-Anhang)
- Stand der Information (Datum, Reichweite)
- Bei aktiver Einwilligung: Mechanismus zur Erfassung und zum Widerruf

### Modus 2: code_review (Auskunfts- und Berichtigungsrecht implementierbar)

Der Server muss technisch unterstützen, dass eine Lehrperson alle über sie gespeicherten Daten abrufen und korrigieren kann.

```bash
grep -rE 'auskunft|access_request|export_personal|self_service' src/
```

**Pass-Pattern:**

```python
@mcp.tool(
    name="exportMyData",
    description="Auskunftsrecht nach Art. 25 revDSG — exportiert alle gespeicherten Daten der anfragenden Lehrperson.",
    annotations={"readOnlyHint": True},
)
async def export_my_data(ctx: Context) -> dict:
    user_claims = verify_token(ctx.request.headers["Authorization"])
    if user_claims["role"] != "lehrperson":
        return {"isError": True, "content": [TextContent(
            type="text", text="Nur Lehrpersonen können diesen Tool verwenden."
        )]}
    user_id = user_claims["sub"]
    return {
        "personal_data": await db.get_all_data_for(user_id),
        "exported_at": datetime.utcnow().isoformat(),
        "access_request_id": str(uuid.uuid4()),
    }
```

### Modus 3: documentation_check (Sozialpartner-Konsultation)

Bei systematischer Verarbeitung im Repo dokumentieren:

```bash
find . -iname '*sozialpartner*' -o -iname '*lch*' -o -iname '*zlv*'
grep -iE 'lch|zlv|berufsverband|sozialpartner' README.md docs/
```

## Pass Criteria

- [ ] Information an Lehrpersonen ist dokumentiert (Wann, wie, Reichweite)
- [ ] Auskunftsrecht ist technisch unterstützt (Tool oder dokumentierter Prozess)
- [ ] Berichtigungsrecht ist technisch unterstützt
- [ ] Bei systematischer Verarbeitung: Konsultation der Berufsverbände dokumentiert
- [ ] Zweckbindung im Verarbeitungsverzeichnis (CH-002) explizit auf Volksschulgesetz-Aufgaben begrenzt
- [ ] Sekundärnutzung (über ursprünglichen Zweck hinaus) wird gesperrt oder geht durch separate Einwilligung

## Common Failures

| Anti-Pattern | Risiko |
|---|---|
| Lehrpersonen werden nicht informiert | Verstoss gegen Informationspflicht (Art. 19 revDSG) |
| Auskunftsrecht «über E-Mail an Datenschutz» — kein technischer Selbstservice | OK rechtlich, aber Operations-Last |
| Server speichert Daten, ohne dass die betroffene Lehrperson das weiss | Schwerwiegender Vertrauensbruch |
| Sekundärnutzung ohne neue Einwilligung | Zweckentfremdung |

## Remediation

### Schritt 1: Information der Lehrpersonen

Eine kurze Mitteilung über Personalreglement-Anhang oder Schulamts-Newsletter:

> Die Stadt Zürich Schulamt setzt seit <Datum> einen MCP-Server zur internen Analyse von <Zweck> ein. Verarbeitete Daten: <Liste>. Rechtsgrundlage: <Verweis>. Auskunftsrecht über das Datenschutz-Postfach datenschutz@zuerich.ch oder via Self-Service unter <URL>.

### Schritt 2: Auskunfts-Tool implementieren

Wie im Pass-Pattern oben.

### Schritt 3: Sozialpartner informieren

Bei systematischer Verarbeitung: Information an LCH-Sektion und ZLV mit Vorlauf von 4–8 Wochen vor Produktiv-Start. Dokumentation der Konsultation im Repo.

## Effort

M — 1–2 Wochen. Information-Workflow + Auskunfts-Tool + Stakeholder-Kommunikation.
