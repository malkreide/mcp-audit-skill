# SEC-001 — Confused Deputy: Per-Client Consent Flow

## Description

Wenn ein MCP-Proxy-Server zwischen einem MCP-Client und einer Drittanbieter-API vermittelt, entsteht ein klassisches «Confused Deputy»-Problem. Der Angriff funktioniert wie folgt:

1. Legitimer Nutzer authentifiziert sich über den Proxy beim Drittanbieter (z.B. GitHub)
2. Drittanbieter setzt Consent-Cookie im Browser, gebunden an die **statische Client-ID des Proxys**
3. Angreifer registriert dynamisch einen bösartigen MCP-Client mit eigener `redirect_uri`
4. Angreifer schickt dem Nutzer einen manipulierten Autorisierungs-Link
5. Browser sendet das legitime Consent-Cookie mit → Drittanbieter überspringt den Consent-Dialog
6. Authorization Code wird an Angreifer-`redirect_uri` weitergeleitet
7. Angreifer hat vollen Zugriff im Namen des Nutzers, **ohne dass dieser jemals einer neuen Berechtigung zugestimmt hat**

Dieser Angriff ist im PDF-Referenz-Material als kritisch markiert und betrifft jede MCP-Architektur mit OAuth-Proxy-Pattern.

## Verification

### Modus 1: code_review (Per-Client Consent UI)

Der Server muss eine **eigene Consent-Seite** vor jedem Upstream-OAuth-Flow rendern. Diese Seite zeigt:
- Den anfragenden MCP-Client eindeutig namentlich
- Die spezifischen API-Scopes
- Die registrierte `redirect_uri` (transparent, nicht versteckt)

```bash
# Suche nach Consent-UI-Implementation
grep -rE "consent|authorize" src/ --include="*.py" --include="*.ts"
find src/ -name "*consent*" -o -name "*authorize*"
```

### Modus 2: code_review (Clickjacking-Schutz)

```bash
# Suche nach X-Frame-Options oder CSP frame-ancestors
grep -rE "X-Frame-Options|frame-ancestors" src/
grep -rE "CSP|Content-Security-Policy" src/
```

**Pass:** `X-Frame-Options: DENY` ODER (bevorzugt) CSP `frame-ancestors 'none'` ist auf der Consent-Seite gesetzt.

### Modus 3: code_review (OAuth State Parameter)

```bash
# Suche nach State-Parameter-Implementation
grep -rE "state.*token|oauth.*state|state.*single.use|nonce" src/
```

**Pass-Kriterien für State:**
- Kryptografisch sicher generiert (`secrets.token_urlsafe()` in Python, `crypto.randomBytes()` in Node)
- Single-Use (nach Validation gelöscht)
- TTL maximal 10 Minuten
- **Erst nach Consent-Bestätigung serverseitig gespeichert** — nicht vorher

### Modus 4: code_review (Cookie-Security)

```bash
# Suche nach Cookie-Setzungen
grep -rE "set_cookie|setCookie|Set-Cookie" src/
```

**Pass-Pattern:**
```python
response.set_cookie(
    key="__Host-mcp-consent",
    value=signed_token,
    secure=True,
    httponly=True,
    samesite="strict",
    path="/",
)
```

**Anforderungen:**
- `__Host-` Prefix (verhindert Subdomain-Override)
- `Secure=True` (nur HTTPS)
- `HttpOnly=True` (kein JS-Zugriff)
- `SameSite=Strict` oder `Lax`
- An spezifische `client_id` gebunden
- Kryptografisch signiert

## Pass Criteria

- [ ] Eigene Per-Client Consent-UI vor Upstream-OAuth-Call vorhanden
- [ ] Consent-Seite ist gegen Clickjacking geschützt (`X-Frame-Options: DENY` oder CSP)
- [ ] OAuth-State-Parameter: kryptografisch sicher, Single-Use, max 10min TTL
- [ ] State wird **erst nach Consent-Bestätigung** serverseitig gespeichert (nicht vor Anfrage)
- [ ] Cookies mit `__Host-` Prefix, `Secure`, `HttpOnly`, `SameSite=Strict|Lax`
- [ ] Cookie ist an spezifische `client_id` gebunden und signiert

## Common Failures

| Anti-Pattern | Konsequenz |
|---|---|
| Direkt-Redirect zu Upstream-OAuth ohne eigene Consent-UI | Confused Deputy direkt ausnutzbar |
| State-Param wird **vor** Consent gespeichert | Consent-Screen kann übersprungen werden |
| State ohne TTL oder Multi-Use | Replay-Angriff möglich |
| Cookies ohne `__Host-` Prefix | Subdomain-Takeover-Angriff erweitert |
| Cookies ohne `SameSite` | CSRF-Angriff möglich |
| Consent-Seite ohne Clickjacking-Schutz | Unsichtbarer iFrame-Trick verleitet Nutzer zu Klick |

## Remediation

### Schritt 1: Eigene Consent-UI implementieren

```python
from fastapi import FastAPI, Request, Form
from fastapi.responses import HTMLResponse, RedirectResponse
import secrets
from itsdangerous import URLSafeTimedSerializer

app = FastAPI()
signer = URLSafeTimedSerializer(SECRET_KEY)

@app.get("/oauth/consent")
async def consent_page(request: Request, client_id: str, scopes: str, redirect_uri: str):
    # Kein State speichern, bis User zustimmt
    return HTMLResponse(
        content=render_consent_template(client_id, scopes, redirect_uri),
        headers={
            "X-Frame-Options": "DENY",
            "Content-Security-Policy": "frame-ancestors 'none'",
        },
    )

@app.post("/oauth/consent/approve")
async def approve_consent(
    request: Request,
    client_id: str = Form(...),
    scopes: str = Form(...),
    redirect_uri: str = Form(...),
):
    # State JETZT generieren und in signiertes Cookie packen
    state = secrets.token_urlsafe(32)
    state_data = {
        "state": state,
        "client_id": client_id,
        "redirect_uri": redirect_uri,
        "scopes": scopes,
    }
    signed = signer.dumps(state_data)

    upstream_url = build_upstream_oauth_url(client_id, scopes, redirect_uri, state)
    response = RedirectResponse(upstream_url)
    response.set_cookie(
        key="__Host-oauth-state",
        value=signed,
        secure=True,
        httponly=True,
        samesite="strict",
        max_age=600,  # 10 min
        path="/",
    )
    return response

@app.get("/oauth/callback")
async def callback(request: Request, code: str, state: str):
    cookie = request.cookies.get("__Host-oauth-state")
    if not cookie:
        raise HTTPException(401, "Missing state cookie")
    try:
        state_data = signer.loads(cookie, max_age=600)
    except Exception:
        raise HTTPException(401, "Invalid or expired state")
    if state_data["state"] != state:
        raise HTTPException(401, "State mismatch")
    # State ist single-use: Cookie löschen
    response = handle_token_exchange(code, state_data)
    response.delete_cookie("__Host-oauth-state")
    return response
```

### Schritt 2: Tests gegen die Angriffsmuster

```python
def test_consent_page_has_clickjacking_protection(client):
    response = client.get("/oauth/consent?client_id=test&scopes=read&redirect_uri=https://...")
    assert response.headers["X-Frame-Options"] == "DENY"

def test_state_cookie_is_host_prefixed(client):
    response = client.post("/oauth/consent/approve", data={...})
    cookie = response.cookies.get("__Host-oauth-state")
    assert cookie is not None
    assert "Secure" in response.headers.get("set-cookie", "")
    assert "HttpOnly" in response.headers.get("set-cookie", "")

def test_state_is_single_use(client):
    # First callback succeeds
    client.get("/oauth/callback?code=xyz&state=abc")
    # Second callback with same state fails
    response = client.get("/oauth/callback?code=xyz&state=abc")
    assert response.status_code == 401
```
