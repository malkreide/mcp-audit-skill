# SEC-023 — DLP auf Tool-Outputs

## Description

HITL-003 (Data Redaction) deckt den Sampling-Pfad ab — PII-Filter vor LLM-Send. SEC-023 ergänzt das auf der **anderen Seite**: bevor Tool-Returns an den LLM-Host gehen, durchlaufen sie ein DLP-Layer (Data Loss Prevention) das nach sensitiven Pattern scannt:

| Datentyp | Pattern | Schweiz-Spezifisch |
|---|---|---|
| AHV-Nummer | `756.XXXX.XXXX.XX` | ja |
| IBAN-CH | `CH\d{2} ...` | ja |
| Kreditkarten | Luhn-validierte 13-19 Digits | nein |
| E-Mail-Adressen | RFC-Pattern | nein |
| Telefonnummern CH | `+41 ...`, `0XX ...` | ja |
| Schüler-IDs | Stadt-Zürich-internes Format | ja |
| Lehrer-Personalnummern | Stadt-Zürich-internes Format | ja |
| Geburtsdaten | DD.MM.YYYY | nein |

Im Unterschied zu HITL-003 (das nur bei Sampling greift) wirkt SEC-023 auf **jeden** Tool-Return — auch bei normalen Read-Tools, die Daten direkt an den LLM-Host senden.

Das Problem: ein Tool, das Verwaltungsdaten zurückgibt, kann durch Daten-Aggregation oder Bug versehentlich PII einschliessen, das im Schema nicht vorgesehen war. Z.B. ein `aggregate_class_size`-Tool, das fälschlicherweise auch die Schüler-Liste mitliefert. Ohne DLP-Layer landet diese PII im LLM-Kontext und potenziell beim LLM-Provider.

Drei Detection-Layer:

1. **Regex-Layer** für strukturierte Patterns (AHV, IBAN, Kreditkarten) — schnell, deterministisch
2. **ML-basiert** mit Microsoft Presidio oder Cloud DLP für unstrukturierte PII (Namen, Adressen, freie-Text-Erwähnungen)
3. **Custom Patterns** für Stadt-Zürich-spezifische IDs (Schüler-Nummern, Klassen-Bezeichner)

Bei Treffer: **redact** (durch Marker ersetzen) oder **block** (Tool-Return als Error). Plus Audit-Log-Eintrag (Synergie zu OBS-005).

## Verification

### Modus 1: code_review (DLP-Middleware aktiv)

```bash
grep -rE 'dlp|presidio|redact|sanitize.*output' src/
```

**Pass-Pattern (Middleware-Wrapper):**

```python
import re
from dataclasses import dataclass
from typing import Callable, Awaitable

# Wiederverwendung der CH-spezifischen Patterns aus HITL-003
from .redaction import CH_PII_RULES, redact

@dataclass
class DLPViolation:
    severity: str  # "low", "medium", "high"
    pattern_name: str
    count: int


def scan_output(content: str) -> list[DLPViolation]:
    violations = []
    for rule in CH_PII_RULES:
        matches = rule.pattern.findall(content)
        if matches:
            violations.append(DLPViolation(
                severity="high" if "AHV" in rule.description else "medium",
                pattern_name=rule.description,
                count=len(matches),
            ))
    return violations


def dlp_protected_tool(tool_func):
    """Wrapper für alle Tools: DLP-Scan auf Output, Redact oder Block bei Treffer."""
    @functools.wraps(tool_func)
    async def wrapper(args, ctx):
        result = await tool_func(args, ctx)
        # Tool-Result serialisieren, scannen, ggf. ersetzen
        result_str = json.dumps(result, default=str)
        violations = scan_output(result_str)

        if violations:
            audit_logger.warning(
                "dlp_violation",
                tool=tool_func.__name__,
                violations=[{"pattern": v.pattern_name, "count": v.count} for v in violations],
            )
            high_violations = [v for v in violations if v.severity == "high"]
            if high_violations:
                # Block: kritische PII darf gar nicht raus
                return {
                    "isError": True,
                    "content": [TextContent(
                        type="text",
                        text=f"Tool output blocked by DLP: contains restricted PII patterns ({len(high_violations)} matches).",
                    )],
                }
            # Medium: redact und durchlassen
            redacted_str, _ = redact(result_str)
            result = json.loads(redacted_str)

        return result
    return wrapper


@dlp_protected_tool
@mcp.tool()
async def aggregate_class_size(args, ctx):
    return await db.aggregate_class_size(args.school_id)
```

async def test_dlp_blocks_ahv_in_tool_output():
    # Mock-DB liefert irrtümlich AHV-Nummer
    with patch.object(db, "get_record") as mock_db:

    result_str = json.dumps(result)
    assert "756.1234.5678.90" not in result_str
    assert "isError" in result or "[AHV-NR]" in result_str


async def test_dlp_audit_log_on_violation():

## Pass Criteria

- [ ] DLP-Middleware-Wrapper auf allen Tool-Returns aktiv
- [ ] Mindestens fünf Pattern-Klassen abgedeckt (AHV, IBAN, Kreditkarte, E-Mail, CH-Telefon)
- [ ] Bei `Verwaltungsdaten` zusätzlich Stadt-Zürich-spezifische IDs (Schüler-Nummern, Lehrer-Personalnummern)
- [ ] High-Severity-Violations werden geblockt (isError), nicht nur redacted
- [ ] Medium-Violations werden redacted und durchgelassen
- [ ] Audit-Log-Eintrag pro Violation (Synergie zu OBS-005)
- [ ] Tests verifizieren Block- und Redact-Verhalten

## Common Failures

| Anti-Pattern | Risiko |
|---|---|
| Kein DLP auf Tool-Outputs | PII-Leak via aggregierte Returns |
| DLP nur Regex, kein ML-Fallback für unstrukturierte PII | Namen, Adressen schlüpfen durch |
| DLP nur bei einigen Tools (Inkonsistenz) | Schwer auditierbarer Schutz |
| Block ohne Audit-Log | Forensik unmöglich, kein Pattern-Tuning |
| Redact bei `data_class == PII` (statt Block) | Datenfluss bleibt unterhalb der Compliance-Schwelle nicht möglich |

## Remediation

### Schritt 1: DLP-Middleware-Modul

Wie Pass-Pattern Modus 1. Wiederverwendung der HITL-003-Pattern-Library für Konsistenz.

### Schritt 2: Per-Tool-Wrapper anwenden

Decorator-Pattern auf alle Tools, die `Verwaltungsdaten` oder `PII` zurückgeben können.

### Schritt 3: Stadt-Zürich-spezifische Patterns

```python
SCHULAMT_PATTERNS = [
    RedactionRule(
        re.compile(r"\bSCH-\d{6,8}\b"),  # hypothetisches Schüler-ID-Format
        "[SCHUELER-ID]",
        "Stadt Zürich Schüler-ID",
    ),
    RedactionRule(
        re.compile(r"\bLP-\d{5}\b"),
        "[LEHRER-ID]",
        "Stadt Zürich Lehrer-Personalnummer",
    ),
]
```

### Schritt 4: SIEM-Alert auf DLP-Violations

```
WHEN COUNT(dlp_violation) > 5 IN 1h FROM SAME tool
THEN alert SECURITY-TEAM
```

## Effort

M — 1–3 Tage. Middleware + Pattern-Library + Tests + Alerts.
