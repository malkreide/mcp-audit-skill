# SEC-005 — DNS-Rebinding-Prevention

## Description

SEC-004 (SSRF-Prevention) verlangt: Resolved IP wird gegen Blocklist geprüft, dann Request mit dieser IP. DNS-Rebinding ist ein verfeinerter Angriff, der diese Defense umgeht — durch zwei verschiedene DNS-Antworten für denselben Hostnamen mit kurzem TTL:

**Ablauf des Angriffs (TOCTOU = Time-Of-Check-Time-Of-Use):**

1. Angreifer kontrolliert `evil.attacker.com` mit DNS-TTL = 1 Sekunde
2. Erste Auflösung: `evil.attacker.com` → `198.51.100.42` (öffentliche IP, passiert SSRF-Check)
3. Server validiert: IP ist nicht in Blocklist → Pass
4. Server macht zweiten DNS-Lookup für eigentlichen Request (typisch in `requests`/`httpx`-Default-Setup)
5. Zweite Auflösung: `evil.attacker.com` → `169.254.169.254` (Cloud-Metadata-IP)
6. Request geht an Cloud-Metadata-Endpoint → Credential-Leak

Mitigation: **DNS-Pinning** — die einmal aufgelöste IP wird für den eigentlichen HTTP-Request verwendet, nicht der Hostname. Das eliminiert die zweite DNS-Auflösung und damit das Rebinding-Fenster.

Im Pass-Pattern von SEC-004 ist DNS-Pinning bereits enthalten (resolved IP → in URL ersetzt). SEC-005 prüft das spezifisch, weil viele SSRF-Implementations DNS-Pinning vergessen und dadurch die SSRF-Defense effektiv unwirksam machen.

## Verification

### Modus 1: code_review (DNS-Resolution einmalig + Pinning)

```bash
grep -rE 'getaddrinfo|gethostbyname|dns\.resolve' src/
grep -rE 'replace\(.*hostname.*resolved|Host:.*hostname' src/
```

**Pass-Pattern:**

```python
import socket
import ipaddress
from urllib.parse import urlparse
import httpx

async def safe_fetch(url: str) -> httpx.Response:
    parsed = urlparse(url)
    if parsed.scheme != "https":
        raise ValueError("Only HTTPS allowed")

    # Step 1: DNS-Lookup einmalig — pinning
    addrinfo = await asyncio.get_event_loop().getaddrinfo(
        parsed.hostname,
        parsed.port or 443,
        type=socket.SOCK_STREAM,
    )

    # Step 2: Range-Check auf resolved IP
    resolved_ip_str = addrinfo[0][4][0]
    resolved_ip = ipaddress.ip_address(resolved_ip_str)
    for blocked in BLOCKED_NETWORKS:
        if resolved_ip in blocked:
            raise ValueError(f"Blocked range: {resolved_ip}")

    # Step 3: Request mit gepinnter IP, Original-Host für TLS-SNI/Cert-Verification
    pinned_url = url.replace(parsed.hostname, resolved_ip_str, 1)
    async with httpx.AsyncClient() as client:
        return await client.get(
            pinned_url,
            headers={"Host": parsed.hostname},  # TLS-SNI braucht Original
            extensions={"sni_hostname": parsed.hostname},
        )
```

**Fail-Pattern (TOCTOU):**

```python
# FAIL: Zwei separate DNS-Lookups
hostname = urlparse(url).hostname
ip = socket.gethostbyname(hostname)  # 1. Lookup für Validation
# Validation passiert hier...
async with httpx.AsyncClient() as client:
    return await client.get(url)  # 2. Lookup, könnte andere IP liefern
```

### Modus 2: code_review (TLS-SNI / Certificate Validation erhalten)

DNS-Pinning bringt eine Komplikation: bei IP-basierter URL würde TLS-Handshake mit IP statt Hostname laufen → Certificate-Verification schlägt fehl. Lösung: `Host`-Header explizit setzen + SNI-Hostname in TLS-Layer behalten.

```bash
grep -rE 'sni_hostname|SSLContext|verify_hostname' src/
```

**Pass:** Original-Hostname wird im `Host`-Header und in SNI-Extension verwendet, IP nur in TCP-Connection-Layer.

### Modus 3: runtime_test (Rebinding-Simulation)

Schwer in Tests zu simulieren ohne DNS-Server-Mock. Alternative: Statisches Test-Pattern mit Mock-Resolver:

```python
from unittest.mock import patch

async def test_dns_pinning_uses_resolved_ip():
    # Resolver gibt erste Antwort
    resolved_ips = ["1.2.3.4", "169.254.169.254"]
    call_count = {"n": 0}

    def mock_getaddrinfo(*args):
        call_count["n"] += 1
        # Würde bei naive Implementation 2x aufgerufen werden
        ip = resolved_ips[min(call_count["n"] - 1, len(resolved_ips) - 1)]
        return [(socket.AF_INET, 0, 0, "", (ip, 443))]

    with patch("socket.getaddrinfo", side_effect=mock_getaddrinfo):
        with pytest.raises(ValueError, match="Blocked range"):
            await safe_fetch("https://evil.example.com/")
        # Sollte nur 1x aufgerufen werden — dann blockiert
        assert call_count["n"] == 1
```

## Pass Criteria

- [ ] DNS-Resolution erfolgt **einmalig** vor dem HTTP-Request
- [ ] Resolved IP wird für die TCP-Connection verwendet (gepinnter URL oder Custom-Resolver)
- [ ] Original-Hostname wird via `Host`-Header und SNI für TLS-Handshake erhalten
- [ ] Certificate-Validation läuft gegen Original-Hostname (nicht IP)
- [ ] Tests verifizieren: nur 1 DNS-Call pro Request

## Common Failures

| Anti-Pattern | Risiko |
|---|---|
| `requests.get(url)` mit Validation davor | Zwei Lookups, TOCTOU |
| `httpx.AsyncClient()` Default ohne Pinning | Implizit zwei Lookups |
| IP in URL, aber kein `Host`-Header | TLS-Cert-Validation schlägt fehl, oft mit `verify=False` umgangen → MITM-Risiko |
| `verify=False` aktiv | Komplette TLS-Sicherheit verloren |

## Remediation

### Schritt 1: HTTP-Client mit Custom Transport

```python
import httpx
import socket
import ipaddress

class PinnedTransport(httpx.AsyncHTTPTransport):
    """HTTPX Transport mit DNS-Pinning."""

    async def handle_async_request(self, request):
        url = request.url
        if url.scheme != "https":
            raise httpx.RequestError("Only HTTPS allowed")

        # Resolve einmalig
        loop = asyncio.get_event_loop()
        addrinfo = await loop.getaddrinfo(
            url.host, url.port, type=socket.SOCK_STREAM
        )
        resolved_ip = addrinfo[0][4][0]

        # Range-Check
        ip = ipaddress.ip_address(resolved_ip)
        for blocked in BLOCKED_NETWORKS:
            if ip in blocked:
                raise httpx.RequestError(f"Blocked IP: {ip}")

        # URL mit gepinnter IP, aber Host-Header bleibt
        pinned_url = httpx.URL(str(url).replace(url.host, resolved_ip, 1))
        new_request = httpx.Request(
            method=request.method,
            url=pinned_url,
            headers=httpx.Headers(request.headers),
            content=request.content,
            extensions=request.extensions,
        )
        new_request.headers["Host"] = url.host
        # SNI bleibt durch URL-Hostname (httpx interner default)
        return await super().handle_async_request(new_request)


# Verwendung
async with httpx.AsyncClient(transport=PinnedTransport()) as client:
    response = await client.get("https://api.external.com/data")
```

### Schritt 2: Alternative — Egress-Proxy

Wenn Custom-Transport zu komplex: Stripe Smokescreen als Sidecar erledigt DNS-Pinning automatisch.

```yaml
# docker-compose.yml
services:
  smokescreen:
    image: stripe/smokescreen:latest
    command: ["--listen-ip", "127.0.0.1", "--listen-port", "4750"]

  mcp-server:
    image: malkreide/mcp-server
    environment:
      HTTPS_PROXY: http://smokescreen:4750
```

```python
# Im Code: einfach Proxy nutzen
async with httpx.AsyncClient(proxy="http://localhost:4750") as client:
    return await client.get(url)
```
