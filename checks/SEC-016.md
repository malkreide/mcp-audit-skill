# SEC-016 — 0.0.0.0-Binding-Prevention (NeighborJack)

## Description

Die empirische Untersuchung von 2025 ergab: ein erheblicher Teil der OSS-MCP-Server bindet ihren HTTP-Listener an `0.0.0.0` (alle Interfaces) und vertraut implizit darauf, dass Firewall-Regeln den Zugang beschränken. Auf einem Entwickler-Laptop in einem öffentlichen WLAN, einem Co-Working-Space oder einer Konferenz wird der lokale MCP-Server damit für **alle** Geräte im selben Subnetz erreichbar.

**Angriff (NeighborJack):**

1. Entwickler startet lokalen MCP-Server für Tests, gebunden an `0.0.0.0:8080`
2. Server läuft mit User-Privilegien, oft mit Zugriff auf Filesystem, API-Keys, lokale Dev-Datenbanken
3. Angreifer im selben Netz scannt mit `nmap -p 8080 192.168.0.0/24`
4. Findet offenen Port, sendet `tools/list`-Request, erhält Tool-Inventar
5. Ruft destruktive Tools auf: liest Files, exfiltriert API-Keys, löscht Daten

**Warum so häufig:** FastMCP, Express, FastAPI haben in vielen Tutorials `host="0.0.0.0"` als Default. Entwickler kopieren das, ohne über Implikationen nachzudenken. In Container-Kontexten ist `0.0.0.0` korrekt (Container-Netzwerk-Isolation), aber für lokale Entwicklung katastrophal.

## Verification

### Modus 1: automated (Code-Suche nach Binding)

```bash
# Python — FastMCP, FastAPI, Uvicorn, Hypercorn
grep -rE 'host\s*=\s*["\x27]0\.0\.0\.0|host\s*=\s*["\x27]\*' src/

# TypeScript / Node — Express, Fastify
grep -rE 'listen\([^,)]*,\s*["\x27]0\.0\.0\.0' src/

# Auch in Konfigurationsdateien
grep -rE '0\.0\.0\.0' . --include="*.toml" --include="*.yaml" --include="*.json" --include="*.env*"
```

### Modus 2: code_review (Deployment-Differenzierung)

Der korrekte Pattern unterscheidet **Container** (wo `0.0.0.0` nötig ist) von **lokal** (wo `127.0.0.1` zwingend).

**Pass-Pattern (Python):**

```python
import os
from fastmcp import FastMCP

mcp = FastMCP("zurich-opendata")

if __name__ == "__main__":
    transport = os.environ.get("MCP_TRANSPORT", "stdio")
    if transport == "stdio":
        mcp.run(transport="stdio")  # kein Network-Binding
    elif transport == "sse":
        # Für lokal: 127.0.0.1, für Container: explizites ENV
        host = os.environ.get("MCP_HOST", "127.0.0.1")
        port = int(os.environ.get("MCP_PORT", "8000"))
        mcp.settings.host = host
        mcp.settings.port = port
        mcp.run(transport="sse")
```

**Pass-Pattern (Dockerfile mit ENV-Override):**

```dockerfile
FROM python:3.11-slim
COPY . /app
WORKDIR /app
RUN pip install --no-cache-dir -e .

# Container muss an 0.0.0.0 binden, sonst kein Netzwerk-Zugriff
# aber explizit, nicht als Default im Code
ENV MCP_TRANSPORT=sse
ENV MCP_HOST=0.0.0.0
ENV MCP_PORT=8000

CMD ["python", "-m", "zurich_opendata_mcp"]
```

**Fail-Pattern:**

```python
# FAIL: 0.0.0.0 als Code-Default
mcp.run(transport="sse", host="0.0.0.0", port=8000)

# FAIL: ohne Differenzierung lokal/container
app.listen(8000, "0.0.0.0", () => console.log("ready"));
```

### Modus 3: runtime_test (Port-Erreichbarkeit von extern)

Auf dem Dev-Laptop:

```bash
# Server starten
python -m my_mcp_server &

# Aus dem selben Netz prüfen — vom anderen Gerät
LOCAL_IP=$(ipconfig getifaddr en0)  # macOS / hostname -I auf Linux
nmap -p 8000 $LOCAL_IP
# Erwartet: 8000/tcp closed (oder filtered)
# Fail: 8000/tcp open
```

## Pass Criteria

- [ ] Im Code ist **kein** `0.0.0.0`-Binding als Default hardcoded
- [ ] Host wird via Environment-Variable konfiguriert, Default ist `127.0.0.1`
- [ ] In Dockerfile ist `MCP_HOST=0.0.0.0` explizit gesetzt — nur dort
- [ ] README erklärt die Differenzierung lokal/container
- [ ] Optional: Bei Start auf `0.0.0.0` ohne Container-Detection wird WARNUNG geloggt

## Common Failures

| Pattern | Risiko |
|---|---|
| `host="0.0.0.0"` als Code-Default | NeighborJack auf Dev-Laptop |
| `--host 0.0.0.0` als Default-CLI-Arg | gleiche Klasse |
| Docker-Compose mit `ports: - "8000:8000"` ohne Bind-Adresse | Container-Port wird auf alle Interfaces des Hosts gebunden |
| Tutorial-Code 1:1 übernommen | FastAPI-Tutorial nutzt `0.0.0.0` ohne Disclaimer |

## Remediation

### Schritt 1: Code-Default auf 127.0.0.1 ändern

```diff
  if __name__ == "__main__":
      transport = os.environ.get("MCP_TRANSPORT", "stdio")
      if transport == "sse":
-         mcp.run(transport="sse", host="0.0.0.0", port=8000)
+         host = os.environ.get("MCP_HOST", "127.0.0.1")
+         port = int(os.environ.get("MCP_PORT", "8000"))
+         mcp.settings.host = host
+         mcp.settings.port = port
+         mcp.run(transport="sse")
```

### Schritt 2: Container-Override im Dockerfile

```dockerfile
ENV MCP_HOST=0.0.0.0
```

### Schritt 3: Docker-Compose Bind-Adresse

```yaml
# docker-compose.yml
services:
  mcp:
    image: malkreide/zurich-opendata-mcp
    ports:
-     - "8000:8000"           # bindet an alle Interfaces
+     - "127.0.0.1:8000:8000" # nur lokal erreichbar
```

### Schritt 4: Warnung bei riskantem Binding

```python
import logging
import socket

def warn_on_dangerous_binding(host: str):
    if host in ("0.0.0.0", "::"):
        # Container-Detection (heuristisch)
        in_container = (
            os.path.exists("/.dockerenv")
            or os.environ.get("KUBERNETES_SERVICE_HOST")
            or os.environ.get("RAILWAY_PROJECT_ID")
        )
        if not in_container:
            logging.warning(
                "Binding to %s outside container context. "
                "This exposes the MCP server to the local network. "
                "Use MCP_HOST=127.0.0.1 for local development.",
                host,
            )
```

### Schritt 5: README-Dokumentation

```markdown
## Network Binding

By default, this server binds to `127.0.0.1` (localhost only).

For production deployment in Docker/Kubernetes, set `MCP_HOST=0.0.0.0`.
**Never** use `0.0.0.0` outside container contexts — it exposes the server
to your local network (NeighborJack risk).
```
