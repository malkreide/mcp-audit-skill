# CH-004 — OGD-CH Lizenz-Compliance

## Description

Die Schweizer Open-Government-Data-Strategie (OGD-CH) verlangt, dass Daten von Bund, Kantonen und Gemeinden unter offenen Lizenzen publiziert werden. Standard ist **CC BY 4.0** (Creative Commons Attribution 4.0 International) oder vergleichbare offene Lizenzen.

Für MCP-Server, die OGD-Daten abrufen und an LLM-Clients weiterleiten, gilt:

1. **Quelle nennen:** Jede Tool-Antwort, die OGD-Daten enthält, muss die Datenquelle angeben (Anbieter + Lizenz)
2. **Lizenz im README:** Der Server muss klar dokumentieren, welche Datenquellen unter welcher Lizenz verwendet werden
3. **Keine Re-Lizenzierung:** Daten können nicht unter restriktivere Lizenzen weitergegeben werden
4. **Provenance erhalten:** Bei Aggregation aus mehreren Quellen muss die Quelle pro Datensatz nachvollziehbar bleiben

OGD-CH-Datenquellen im Schweizer Kontext:
- opendata.swiss (zentrales Portal Bund)
- statistik.zh.ch (Kanton Zürich)
- data.stadt-zuerich.ch (Stadt Zürich)
- meteoswiss / MétéoSuisse
- BFS Bundesamt für Statistik
- Curia Vista (parlament.ch)

## Verification

### Modus 1: code_review (Source-Field in Tool-Returns)

```bash
# Suche nach Source/Provenance-Feldern in Tool-Returns
grep -rE '"source"|"license"|"provenance"|"attribution"' src/
```

**Pass-Pattern:**

```python
from pydantic import BaseModel, Field
from typing import Literal

class OGDResponse(BaseModel):
    source: str = Field(
        default="data.stadt-zuerich.ch — Stadt Zürich, CC BY 4.0"
    )
    license: Literal["CC BY 4.0", "CC0", "CC BY-SA 4.0"] = "CC BY 4.0"
    provenance: Literal["live_api", "cached", "weekly_dump"]
    retrieved_at: str
    results: list[dict]

@mcp.tool()
async def search_education_stats(query: str, ctx) -> OGDResponse:
    raw = await api.search(query)
    return OGDResponse(
        provenance="live_api",
        retrieved_at=datetime.utcnow().isoformat(),
        results=raw,
    )
```

### Modus 2: documentation_check (README-Lizenz-Sektion)

```bash
grep -iE 'cc by|creative commons|opendata|ogd|license' README.md README.de.md
```

**Pass-Pattern (README):**

```markdown
## Datenquellen & Lizenzen

Dieser Server nutzt folgende Open Government Data Quellen:

| Quelle | Lizenz | Attribution-Text |
|---|---|---|
| Curia Vista | CC BY 4.0 | © Schweizer Parlament, CC BY 4.0 |
| data.stadt-zuerich.ch | CC BY 4.0 | © Stadt Zürich, CC BY 4.0 |
| BFS — STAT-TAB | CC BY 4.0 | © Bundesamt für Statistik, CC BY 4.0 |

Tool-Antworten enthalten ein `source`-Feld mit der Attribution-Information.
Die Daten werden ohne Modifikation weitergegeben (sofern nicht in der
Tool-Description anders dokumentiert, z.B. bei Aggregations-Tools).
```

### Modus 3: code_review (keine Lizenz-Konflikte)

Bei Aggregation aus mehreren Quellen: Lizenzen müssen kompatibel sein. CC BY 4.0 + CC BY-SA 4.0 in einem aggregierten Output → Output muss CC BY-SA 4.0 sein (Share-Alike-Vererbung).

```bash
# Falls verschiedene Lizenz-Werte vorkommen
grep -rE 'CC BY-SA|CC BY-NC|CC BY-ND' src/
```

## Pass Criteria

- [ ] Tool-Antworten enthalten `source`-Feld mit Quelle und Lizenz
- [ ] README dokumentiert alle verwendeten Datenquellen mit Lizenzen
- [ ] Bei Aggregation: Provenance pro Datensatz erhalten (nicht nur global)
- [ ] Keine Lizenz-Konflikte (z.B. CC BY-NC mit kommerzieller Verwendung)
- [ ] Attribution-Text genau gemäss Lizenz-Vorgabe (z.B. CC BY 4.0 verlangt: Autor, Quelle, Lizenz, ggf. Modifikation)

## Common Failures

| Anti-Pattern | Risiko |
|---|---|
| Tool-Antwort ohne `source`-Feld | Lizenz-Verstoss (BY = Attribution-Pflicht) |
| README nennt Datenquelle, aber nicht Lizenz | Unklare Lizenz-Lage für Nutzer |
| CC BY-SA gemischt mit CC BY ohne klare Markierung | Re-Lizenzierungs-Problem |
| Daten modifiziert (Aggregation) ohne Hinweis | CC BY 4.0 verlangt Modifikations-Hinweis |

## Remediation

### Schritt 1: Provenance-Envelope einführen

Wie im Pass-Pattern oben — `OGDResponse` als Standard-Return-Type für alle OGD-Tools.

### Schritt 2: README-Sektion ergänzen

Standardvorlage:

```markdown
## Datenquellen & Lizenzen

| Quelle | URL | Lizenz | Attribution |
|---|---|---|---|
| <Name> | <URL> | <Lizenz-Code> | <Genauer Attribution-Text> |
```

### Schritt 3: Bei Aggregation: Per-Record-Provenance

```python
class AggregatedResult(BaseModel):
    summary: dict
    source_breakdown: list[dict]  # je Datensatz: source + license
```
